# 📰 Конструктор дайджеста «Главные новости КОС» v2

Локальное приложение для одного человека, которое автоматически собирает корпоративный дайджест из постов за 2 недели. Использует LLM (OpenAI GPT-4) для классификации, отбора главных новостей, написания заголовков-каламбуров. Результат — `.htm`-файл, который копируется в Outlook одним движением.

## Что нового по сравнению с v1

| Что | Было | Стало |
|---|---|---|
| Скорость генерации | ~5–10 минут | ~1–2 минуты |
| Стоимость на выпуск | ~$0.50 | ~$0.10 (батчинг + кэш) |
| Повторный запуск | Платный | Бесплатный (кэш) |
| Параллелизм LLM | Нет | 4 параллельных потока |
| Сбои LLM | Молча теряют посты | Retry × 3 + warning |
| Невалидный Excel | Stacktrace | Понятная ошибка |
| Перегенерация одной карточки | Нет | Кнопка 🎲 |
| Замена фото в карточке | Нет | Выпадающий список |
| Порядок рубрик | Случайный | Строго канонический |
| Цитата в рубрике | Иногда не там | Гарантированно где надо |
| Видео-блок | После первой попавшейся | Всегда после ПРОИЗВОДСТВА |
| Image_file терялся | Да (баг merge) | Защищено |

## Установка (один раз)

### Шаг 1. Проверить Python

Открыть командную строку (`Win + R` → введите `cmd` → Enter), ввести:

```
python --version
```

Нужна версия **3.10 или выше**. Если её нет — установить с https://www.python.org/downloads/
**Важно:** при установке поставьте галочку **«Add Python to PATH»**.

### Шаг 2. Распаковать архив

Распакуйте `kos_digest_v2.zip` в удобную папку, например:
```
C:\Users\Ваше_Имя\Documents\kos_digest\
```

### Шаг 3. Установить зависимости

В командной строке перейдите в папку проекта и установите библиотеки:

```
cd C:\Users\Ваше_Имя\Documents\kos_digest
pip install -r requirements.txt
```

Установка займёт 1–2 минуты.

### Шаг 4. Указать API-ключ

1. В корне проекта найдите файл `.env.example`
2. Сделайте его копию и переименуйте в `.env` (без расширения `.example`)
3. Откройте `.env` в Блокноте
4. Впишите ключ от LLM-посредника:

```
OPENAI_API_KEY=sk-ваш_ключ_тут
OPENAI_BASE_URL=https://api.proxyapi.ru/openai/v1
OPENAI_MODEL=gpt-4o
```

**Примеры base_url для популярных посредников:**
- ProxyAPI: `https://api.proxyapi.ru/openai/v1`
- VseGPT: `https://api.vsegpt.ru/v1`
- AITunnel: `https://api.aitunnel.ru/v1/`
- OpenAI напрямую: `https://api.openai.com/v1`

### Шаг 5. Smoke-test

Проверим, что всё установлено правильно:

```
python test_smoke.py
```

Если в конце выведено `🎉 OK — приложение работает` — всё готово. Можно работать.

## Использование (каждый выпуск)

### Шаг 1. Подготовить Excel с постами

В папке `sample_input/` есть `posts.xlsx` с шапкой. По его аналогии заполните данные:

| Колонка | Что | Обязательная? | Пример |
|---|---|---|---|
| `date` | Дата публикации | нет | `2026-05-09` |
| `author` | Автор/спикер | нет | `Татьяна Щербакова, химик ЦЗЛ` |
| `title` | Заголовок поста с портала | нет | `Дефекты в полиэтилене` |
| `text` | **Полный текст поста** | **да** | (целиком, как на портале) |
| `link` | Ссылка на портал | нет | `https://social.sibur.ru/log/298137` |
| `image_file` | **Имя файла фото** | нет | `PHOTO-12345.jpg` |

**Совет:** удобно заполнять по очереди — открыли портал, скопировали текст, ссылку, сохранили фото в папку, написали имя в `image_file`. На 30 постов уходит 30–40 минут.

### Шаг 2. Подготовить папку с фотографиями

Сложите все фотографии в одну папку, например:
```
C:\Users\Ваше_Имя\Pictures\digest_may\
```

Имена должны **в точности** совпадать с тем, что вы написали в `image_file` (расширение учитывается). Регистр (большие/маленькие буквы) не важен.

### Шаг 3. Запустить приложение

В командной строке:
```
cd C:\Users\Ваше_Имя\Documents\kos_digest
streamlit run app.py
```

Откроется браузер с интерфейсом. Если не открылся сам — скопируйте адрес из консоли (обычно `http://localhost:8501`).

### Шаг 4. Сгенерировать дайджест

В боковой панели слева:

1. **Загрузите Excel** с постами (кнопка «Browse files»)
2. **Укажите путь** к папке с фотографиями (полный путь, например `C:\Users\Имя\Pictures\digest_may`)
3. **Укажите папку** для сохранения результата (по умолчанию `output/`)
4. **Выберите дату** выпуска
5. **Кэш** — оставьте включённым (так быстрее)
6. **Debug-режим** — включите при первом запуске, чтобы видеть промежуточные данные
7. Нажмите **🚀 Сгенерировать черновик**

Через 1–2 минуты внизу появится превью дайджеста.

### Шаг 5. Проверить и поправить

В превью каждый блок можно развернуть кнопкой «Редактировать»:
- **Главные новости** — изменить заголовок, заменить фото
- **Главная цифра** — поменять число и описание
- **Главное видео** — заголовок, описание, обложку
- **Цитата** — текст, автора, должность, фото, в какую рубрику ставить
- **Карточки рубрик** — заголовок (КАПСОМ), текст, фото, флаг «с фото / без»

Для карточки рубрики есть **🎲 Перегенерировать** — LLM придумает новый заголовок-каламбур и текст. Используйте, если изначальный получился пресный.

### Шаг 6. Выгрузить .htm

Внизу страницы нажмите **📥 Собрать .htm**. В папке `output/` появится:
```
digest_2026-05-12.htm           ← главный файл
digest_2026-05-12.files\         ← папка с картинками
    banner.png
    main_PHOTO-12345.jpg
    card_PHOTO-67890.jpg
    rubric_production.png
    ...
```

### Шаг 7. Вставить в Outlook

1. Откройте `digest_2026-05-12.htm` в браузере (двойной клик)
2. Выделите всё (`Ctrl+A`)
3. Скопируйте (`Ctrl+C`)
4. В Outlook создайте новое письмо, поставьте курсор в тело и вставьте (`Ctrl+V`)
5. Картинки подтянутся автоматически (они в папке `.files` рядом)

## Тонкая настройка

### Поправить фирменный тон

В файле `prompts/rewrite.txt` есть **6 примеров хороших карточек** (Пример 1–6). Если фирменный тон со временем меняется (новые типичные заголовки), просто допишите туда новые примеры.

### Поправить критерии отбора

В `prompts/plan.txt` — правила, как отбирать главные новости, цитату, видео. Можно менять.
В `prompts/classify.txt` — правила распределения по рубрикам.

### Поменять модель LLM

В `.env`:
- `gpt-4o` — лучший баланс (рекомендую)
- `gpt-4o-mini` — в 10 раз дешевле, но заголовки получаются попроще
- `gpt-4-turbo` — старая, дорогая, не используйте

### Поменять размер картинок

В `src/config.py`:
```python
SIZE_MAIN = (286, 161)    # карточка в Главном
SIZE_CARD = (265, 176)    # карточка рубрики
```

### Очистить кэш

Если хочется заставить LLM придумать всё заново на тех же данных:
- Кнопка **🗑 Очистить кэш LLM** в сайдбаре, ИЛИ
- Удалить папку `.cache/` руками

### Отключить параллелизм (если ваш посредник лимитирует RPM)

В `src/config.py`:
```python
LLM_PARALLEL_WORKERS = 1  # вместо 4
```

## Что делать, если...

**…ошибка «API-ключ не задан»**
Проверьте, что есть файл `.env` (не `.env.example`), и в нём прописан настоящий ключ, а не `sk-замените-...`.

**…ошибка «Не удалось открыть Excel»**
Закройте файл в Excel (он держит блокировку) и запустите ещё раз.

**…фотографии не появляются в готовом дайджесте**
- Проверьте, что в Excel в колонке `image_file` стоят имена файлов из вашей папки.
- Проверьте, что путь к папке с фото указан правильно в боковой панели.
- Включите Debug-режим, посмотрите `output/draft_debug.json` — там видно, доехал ли `image_file` до этапа сборки.

**…LLM выбрала не ту новость в главные**
Откройте раздел «Главное» в превью, нажмите «Редактировать» на карточке, поменяйте заголовок. Если нужно поменять сам пост — отредактируйте `plan.txt` и сделайте новый запуск.

**…карточек в рубрике мало (1–2)**
Возможно, у вас в выпуске мало постов нужной тематики. Дайджест соберётся как есть. Можно либо добавить посты в Excel, либо отредактировать промпт `plan.txt` (правило «в рубрике 2–5 карточек»).

**…цитата пустая, хочу добавить вручную**
В блоке «Цитата» есть кнопка **➕ Добавить цитату вручную**. Откроется форма для редактирования.

**…хочу другой порядок главных новостей**
Поменяйте местами заголовки руками через «Редактировать».

**…LLM пишет слишком сухие заголовки**
- Нажмите 🎲 «Перегенерировать» — будет вариант с другой температурой
- Допишите в `prompts/rewrite.txt` ваши любимые примеры заголовков из старых выпусков

**…счёт за LLM большой**
- Включите кэш (`LLM_CACHE_ENABLED=1` в `.env`)
- Используйте модель `gpt-4o-mini` (в 10 раз дешевле)
- Не запускайте дайджест на 100+ постах за раз — фильтруйте посты заранее

**…streamlit не открывает браузер**
Скопируйте адрес из командной строки (обычно `http://localhost:8501`) и откройте сами.

## Что под капотом

### Пайплайн обработки

```
Excel (30–50 постов)
       │
       ▼
ШАГ 1: Классификация
  - 4 батча по 10 постов параллельно
  - Каждый пост получает: рубрику, важность, цитату, цифру, метки
       │
       ▼
ШАГ 2: План дайджеста
  - Один большой LLM-вызов
  - Решает: что в Главное, что в цифру, цитату, видео
  - Распределяет остальное по рубрикам
       │
       ▼
ШАГ 3: Перепись карточек
  - Параллельно (4 потока)
  - Каждая карточка: заголовок-каламбур + текст 2–5 строк
       │
       ▼
Сборка HTML
  - Кадрирование фото под слоты (286×161, 265×176, ...)
  - Копирование в .files-папку
  - Рендер Jinja2-шаблона
       │
       ▼
готовый .htm для Outlook
```

### Защита от типовых проблем

- **Кэш LLM:** sha1-хеш от промпта → JSON-ответ. Повторный запуск на тех же данных = 0₽
- **Retry:** 3 попытки с экспоненциальным бэкоффом при ошибках сети/парсинга
- **Параллелизм:** ThreadPoolExecutor с 4 потоками (настраивается)
- **Защита image_file:** в merge `{**classification, **post}` исходные поля Excel всегда побеждают
- **Канонический порядок рубрик:** всегда Производство → ПСС → Безопасность → Забота → Карьера → События → Достижения
- **Сингтоны:** один пост может попасть только в один блок (Главное / Цифра / Цитата / Видео / Рубрика)
- **Видео ставится после ПРОИЗВОДСТВА:** жёстко, не «после первой рубрики»
- **Валидация Excel:** понятные сообщения вместо stacktrace
- **Валидация фотопапки:** проверка, что файлы из image_file существуют
